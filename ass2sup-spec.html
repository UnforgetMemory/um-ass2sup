1 项目概述

1.1 目标

构建一个 100% Rust 实现 的命令行工具，将 ASS/SSA/SRT 字幕文件转换为蓝光兼容的 PGS (Presentation Graphic Stream) SUP 文件。无需外部依赖（无 AviSynth、无 Java、无 DLL），跨平台运行。

1.4 用户界面

CLI-first 设计，支持以下用法：

# 基础用法
ass2sup input.ass -o output.sup

# 带选项
ass2sup input.ass -o output.sup \
  --resolution 1920x1080 \
  --fps 23.976 \
  --validate \
  --overlap-warn \
  --karaoke-mode split \
  --quantizer median-cut \
  --verbose

# SRT 输入
ass2sup input.srt -o output.sup --font "Arial" --font-size 48

# 批量处理
ass2sup *.ass --output-dir ./sup/ --parallel

2.3 设计原则

🔌 模块化

每个 crate 职责单一，可独立测试、独立复用。ass-parser 可被其他项目引用，pgs-encoder 可独立使用。

⚡ 零拷贝

解析阶段使用 &str 引用原始文本，避免不必要的分配。渲染阶段使用 arena 分配器管理帧缓冲区。

🛡️ 类型安全

时间戳使用新类型 Timestamp(u64)（毫秒），颜色使用 Rgba(u8,u8,u8,u8)，杜绝类型混淆。

🔄 流式处理

采用 iterator/stream 模式，逐帧处理而非全量加载，支持处理数万条字幕的大型文件。

3.2 各阶段详细说明

Stage 1: 解析 (Parse)

输入: 原始文本文件 (UTF-8/UTF-16/GBK)

输出: SubtitleDocument AST

struct SubtitleDocument {
    script_info: ScriptInfo,
    styles: Vec<Style>,
    events: Vec<Event>,
    attachments: Vec<Attachment>,  // 字体文件等
}

struct Event {
    layer: u32,
    start: Timestamp,
    end: Timestamp,
    style_name: String,
    text: String,
    parsed_tags: Vec<OverrideTag>,  // 预解析的 override tags
    karaoke_segments: Option<Vec<KaraokeSegment>>,
}

关键行为:

  • 自动检测编码 (BOM / chardet)
  • 预解析所有 override tags 为结构化类型
  • 识别并标记 karaoke 段
  • 容错处理：跳过无法解析的行，收集 warnings

Stage 2: 校验 (Validate)

输入: SubtitleDocument

输出: ValidationReport（warnings + errors）

详见 Section 6: 校验机制

Stage 3: 渲染 (Render)

输入: SubtitleDocument + 目标分辨率 + 帧率

输出: 帧序列 Vec<RenderedFrame>

struct RenderedFrame {
    pts: u64,           // Presentation Timestamp (90kHz)
    duration: u64,      // 显示时长 (90kHz)
    width: u16,
    height: u16,
    bitmap: Vec<u8>,    // RGBA pixels
    windows: Vec<Window>, // 支持多窗口（字幕堆叠时）
}

渲染管线:

  1. 字体匹配与加载 (fontdb)
  2. 文本 shaping (rustybuzz)
  3. 字形光栅化 (ttf-parser + tiny-skia)
  4. 应用特效 (blur, border, shadow, rotation)
  5. Alpha 合成 (premultiplied alpha)
  6. Karaoke 效果叠加

Stage 4: 量化 (Quantize)

输入: RGBA 帧

输出: 索引色帧 (8-bit palette + indices)

PGS 格式限制每帧最多 256 色。使用 Median Cut 或 Octree 算法进行颜色量化，配合 Floyd-Steinberg 抖动减少色带。

struct QuantizedFrame {
    palette: [Rgba; 256],      // 最多 256 色
    indices: Vec<u8>,          // 每像素调色板索引
    transparent_index: u8,     // 透明色索引
}

Stage 5: PGS 编码 (Encode)

输入: 量化帧序列

输出: PGS SUP 二进制流

详见 Section 7: PGS 编码

特效渲染实现

特效实现方式复杂度
\pos(x,y) 直接设置渲染偏移 低
\move(x1,y1,x2,y2) 按时间插值计算当前位置 中
\fad(in,out) Alpha 通道按时间渐变 低

\blur(n) 高斯模糊 (tiny-skia 或自实现) 中
\bord(n) SDF (Signed Distance Field) 描边 中
\shad(x,y) 偏移 + 模糊的阴影层 低
\frz(angle) 2D 旋转矩阵变换 中
\fscx, \fscy 缩放变换 低
\clip(x1,y1,x2,y2) 矩形裁剪 (alpha mask) 低
\clip(矢量) 矢量路径裁剪 (tiny-skia clip) 高
\t(tag) 按时间插值属性值 高
\p(绘图模式) ASS 绘图指令 → tiny-skia Path 高

渲染流程（单帧）

fn render_frame(events: &[&Event], time: Timestamp, config: &RenderConfig) -> RenderedFrame {
    let mut frame = FrameBuffer::new(config.width, config.height);

    // 按 layer 排序
    let mut sorted: Vec<_> = events.iter().collect();
    sorted.sort_by_key(|e| e.layer);

    for event in sorted {
        let ctx = build_render_context(event, time, config);

        // 1. Text layout (shaping)
        let glyphs = shape_text(&ctx, config);

        // 2. Rasterize glyphs
        let bitmaps = rasterize_glyphs(&glyphs, &ctx);

        // 3. Apply effects (outline, shadow, blur)
        let composited = apply_effects(&bitmaps, &ctx);

        // 4. Apply karaoke highlighting if applicable
        let final_bitmap = if let Some(phase) = &ctx.karaoke_phase {
            apply_karaoke(&composited, phase, &ctx)
        } else {
            composited
        };

        // 5. Apply transformations (rotation, scale)
        let transformed = apply_transforms(&final_bitmap, &ctx);

        // 6. Composite onto frame (premultiplied alpha blending)
        frame.composite(&transformed, ctx.pos_x, ctx.pos_y);
    }

    frame.to_rendered_frame(time)
}

  • 完全透明像素跳过：alpha=0 的像素不参与量化计算

5.1 Karaoke 支持

🎤 Karaoke 渲染策略

Karaoke Tag 类型

Tag行为渲染效果
{\k d} 瞬间切换 — 持续 d/100 秒后高亮 音节到时间点立即变色
{\kf d} 填充 — 从左到右平滑扫过 渐变高亮从左到右覆盖音节
{\ko d} 轮廓 — 音节开始时轮廓闪烁 开始时短暂轮廓高亮
{\kt d} 精确时间 — 使用绝对时间 与 \k 类但使用绝对时间

渲染方案

/// Karaoke 渲染阶段
#[derive(Debug, Clone)]
pub enum KaraokePhase {
    /// 未开始 — 显示为次要颜色
    Pending,
    /// 活跃 — 正在高亮
    Active { progress: f64 },  // 0.0..=1.0
    /// 完成 — 显示为主要颜色
    Done,
}

/// Karaoke 帧渲染
fn render_karaoke_segment(
    segment: &KaraokeSegment,
    time: Timestamp,
    ctx: &RenderContext,
) -> KaraokePhase {
    let seg_start = /* segment absolute start time */;
    let seg_end = seg_start + Duration::from_millis(segment.duration);

    if time < seg_start {
        KaraokePhase::Pending
    } else if time >= seg_end {
        KaraokePhase::Done
    } else {
        let progress = (time - seg_start).as_secs_f64()
                     / segment.duration as f64;
        KaraokePhase::Active { progress }
    }
}

/// 根据 karaoke phase 调整颜色
fn karaoke_color(phase: &KaraokePhase, ctx: &RenderContext) -> Rgba {
    match phase {
        KaraokePhase::Pending => ctx.secondary_color,
        KaraokePhase::Done => ctx.primary_color,
        KaraokePhase::Active { progress } => {
            // 对于 \kf: 从 secondary 渐变到 primary
            lerp_color(ctx.secondary_color, ctx.primary_color, *progress)
        }
    }
}

双层渲染策略（Karaoke）

关键设计: Karaoke 字幕需要双层渲染——底层显示完整文本（次要颜色），顶层显示当前音节（主要颜色）。对于 \kf 模式，顶层使用 clip 实现渐变扫描效果。 
fn render_with_karaoke(
    event: &Event,
    time: Timestamp,
    frame: &mut FrameBuffer,
) {
    let segments = event.karaoke.as_ref().unwrap();

    // Layer 1: 完整文本（次要颜色 / 已完成音节用主要颜色）
    for seg in segments {
        let phase = render_karaoke_segment(seg, time, event);
        let color = karaoke_color(&phase, &ctx);
        render_text_segment(seg, color, frame);
    }

    // Layer 2: 当前活跃音节的高亮效果
    if let Some(active) = find_active_segment(segments, time) {
        match active.style {
            KaraokeStyle::Fill => {
                // 使用矩形 clip 实现从左到右扫描
                let progress = get_progress(active, time);
                let clip_x = segment_start_x + segment_width * progress;
                render_with_clip(active, 0.0, 0.0, clip_x, height, frame);
            }
            KaraokeStyle::Outline => {
                // 轮廓闪烁效果
                render_with_outline_highlight(active, frame);
            }
            _ => { /* \k, \kt: 直接变色，无需额外层 */ }
        }
    }
}

/// Override tag 解析器 — 支持嵌套
pub fn parse_override_tags(input: &str) -> Result<Vec<OverrideTag>, ParseError> {
    let mut tags = Vec::new();
    let mut chars = input.char_indices().peekable();

    while let Some((i, ch)) = chars.next() {
        match ch {
            '\\' => {
                let tag_name = consume_tag_name(&mut chars);
                let tag_value = consume_tag_value(&mut chars);
                let tag = match tag_name.as_str() {
                    "pos" => parse_pos(&tag_value)?,
                    "move" => parse_move(&tag_value)?,
                    "fad" => parse_fad(&tag_value)?,
                    "fade" => parse_fade(&tag_value)?,
                    "t" => parse_transform(&tag_value)?,
                    "k" | "K" => parse_karaoke(&tag_name, &tag_value)?,
                    // ... 其他标签
                    _ => OverrideTag::Unknown {
                        tag: tag_name, value: tag_value
                    },
                };
                tags.push(tag);
            }
            '{' => { /* 进入嵌套 */ }
            '}' => { /* 退出嵌套 */ }
            _ => { /* 普通文本字符 */ }
        }
    }
    Ok(tags)
}

/// 检测事件列表中的所有重叠
pub fn detect_overlaps(
    events: &[Event],
    config: &OverlapConfig,
) -> Vec<OverlapWarning> {
    let mut warnings = Vec::new();

    // 1. 构建时间线区间树 (Interval Tree) — O(n log n)
    let tree = build_interval_tree(events);

    // 2. 对每个事件，查询重叠区间
    for (i, event) in events.iter().enumerate() {
        let overlapping = tree.query_overlapping(event.start, event.end);

        for j in overlapping {
            if j <= i { continue; }  // 避免重复报告

            let other = &events[j];

            // 3. 层级过滤
            if !config.check_cross_layer && event.layer != other.layer {
                continue;
            }

            // 4. 视觉重叠检测（可选）
            if config.check_visual_overlap {
                if !may_visually_overlap(event, other) {
                    continue;
                }
            }

            // 5. 计算重叠信息
            let overlap_start = event.start.max(other.start);
            let overlap_end = event.end.min(other.end);
            let overlap_duration = overlap_end - overlap_start;

            warnings.push(OverlapWarning {
                event_a_idx: i,
                event_b_idx: j,
                overlap_start,
                overlap_duration,
                severity: classify_severity(overlap_duration, config),
                suggestion: generate_suggestion(event, other),
            });
        }

        // 6. 间隔过短检测
        let nearest_after = find_nearest_after(events, i);
        if let Some((j, gap)) = nearest_after {
            if gap < Duration::from_millis(config.min_gap_ms) {
                warnings.push(OverlapWarning {
                    event_a_idx: i,
                    event_b_idx: j,
                    overlap_start: event.end,
                    overlap_duration: Duration::ZERO,
                    severity: Severity::Info,
                    suggestion: Some(format!(
                        "间隔仅 {}ms，视觉上可能产生堆叠感",
                        gap.as_millis()
                    )),
                });
            }
        }
    }

    warnings
}

/// 视觉重叠判断 — 基于对齐方式和位置
fn may_visually_overlap(a: &Event, b: &Event) -> bool {
    // 如果两者都有 \pos，比较坐标
    if let (Some(pos_a), Some(pos_b)) = (get_pos(a), get_pos(b)) {
        let distance = ((pos_a.x - pos_b.x).powi(2)
                      + (pos_a.y - pos_b.y).powi(2)).sqrt();
        return distance < 100.0;  // 阈值可配置
    }

    // 如果没有 \pos，根据 alignment 和 margin 推断位置
    let rect_a = estimate_bounding_rect(a);
    let rect_b = estimate_bounding_rect(b);
    rect_a.intersects(&rect_b)
}

7.3 RLE 编码

📦 PGS RLE (Run-Length Encoding)

PGS 使用特定的 RLE 变体压缩像素数据：

/// PGS RLE 编码器
pub fn rle_encode(palette_indices: &[u8], width: u32, height: u32) -> Vec<u8> {
    let mut output = Vec::new();

    for y in 0..height {
        let row = &palette_indices[(y * width) as usize..((y + 1) * width) as usize];
        let mut x = 0;

        while x < width as usize {
            let color = row[x];

            // 计算连续相同像素的数量
            let mut run_length = 1;
            while x + run_length < width as usize
                && row[x + run_length] == color
                && run_length < 0x3FFF  // 最大 RLE 长度
            {
                run_length += 1;
            }

            // 编码
            if run_length == 1 {
                // 单像素：直接输出
                if color == 0 {
                    // 透明像素特殊处理
                    output.push(0x00);
                } else {
                    output.push(color);
                }
            } else if run_length <= 0x3F {
                // 短行程：1 字节长度
                let byte1 = 0x00 | ((run_length >> 8) as u8 & 0x03);
                let byte2 = (run_length & 0xFF) as u8;
                if color == 0 {
                    output.push(byte1);
                    output.push(byte2);
                } else {
                    output.push(color);
                    output.push(byte1 | 0x40);
                    output.push(byte2);
                }
            } else {
                // 长行程：2 字节长度
                let byte1 = 0x00 | ((run_length >> 8) as u8 & 0x3F);
                let byte2 = (run_length & 0xFF) as u8;
                if color == 0 {
                    output.push(byte1 | 0x40);
                    output.push(byte2);
                } else {
                    output.push(color);
                    output.push(byte1 | 0x80);
                    output.push(byte2);
                }
            }

            x += run_length;
        }

        // 行结束标记
        if y < height - 1 {
            output.push(0x00);  // end of line
        }
    }

    output
}

7.4 YCbCr 颜色转换

/// RGBA → YCbCr 转换 (BT.601)
pub fn rgba_to_ycbcr(r: u8, g: u8, b: u8) -> (u8, u8, u8) {
    let r = r as f64;
    let g = g as f64;
    let b = b as f64;

    let y  = ( 0.299 * r + 0.587 * g + 0.114 * b).round() as u8;
    let cb = (-0.169 * r - 0.331 * g + 0.500 * b + 128.0).round() as u8;
    let cr = ( 0.500 * r - 0.419 * g - 0.081 * b + 128.0).round() as u8;

    (y, cb, cr)
}

/// 调色板条目构建
pub fn build_palette(quantized: &QuantizedFrame) -> Vec<PaletteEntry> {
    quantized.palette.iter().enumerate().map(|(i, rgba)| {
        let (y, cb, cr) = rgba_to_ycbcr(rgba.r, rgba.g, rgba.b);
        PaletteEntry {
            index: i as u8,
            y, cb, cr,
            alpha: rgba.a,
        }
    }).collect()
}

7.5 Display Set 构造

/// 一帧字幕 = 一个 Display Set
pub fn build_display_set(
    frame: &QuantizedFrame,
    pts: u64,
    object_id: u16,
    window_id: u8,
    composition_number: u16,
) -> Vec<Segment> {
    let mut segments = Vec::new();

    // 1. PCS — 定义显示合成
    segments.push(Segment {
        segment_type: SegmentType::Pcs,
        pts,
        dts: pts,  // 对于字幕，DTS 通常等于 PTS
        payload: SegmentPayload::Pcs(PcsPayload {
            width: frame.width as u16,
            height: frame.height as u16,
            frame_rate: 0x10,  // 根据实际 FPS 设置
            composition_number,
            composition_state: CompositionState::EpochStart,
            palette_update: false,
            palette_id: 0,
            num_objects: 1,
            compositions: vec![ObjectComposition {
                object_id,
                window_id,
                cropped: false,
                forced: false,
                x: 0, y: 0,
                crop_x: 0, crop_y: 0,
                crop_w: 0, crop_h: 0,
            }],
        }),
    });

    // 2. WDS — 定义窗口
    segments.push(Segment {
        segment_type: SegmentType::Wds,
        pts,
        dts: pts,
        payload: SegmentPayload::Wds(WdsPayload {
            num_windows: 1,
            windows: vec![WindowDef {
                window_id,
                x: 0, y: 0,
                width: frame.width as u16,
                height: frame.height as u16,
            }],
        }),
    });

    // 3. PDS — 定义调色板
    segments.push(Segment {
        segment_type: SegmentType::Pds,
        pts,
        dts: pts,
        payload: SegmentPayload::Pds(PdsPayload {
            palette_id: 0,
            version: 0,
            entries: build_palette(frame),
        }),
    });

    // 4. ODS — 定义对象（可能分多个段，如果 RLE 数据很大）
    let rle = rle_encode(&frame.indices, frame.width, frame.height);
    let chunks = chunk_rle_data(&rle, 0xFFE0);  // 每段最大 ~64KB
    for (i, chunk) in chunks.iter().enumerate() {
        segments.push(Segment {
            segment_type: SegmentType::Ods,
            pts,
            dts: pts,
            payload: SegmentPayload::Ods(OdsPayload {
                object_id,
                object_version: 0,
                last_in_sequence: i == chunks.len() - 1,
                width: frame.width as u16,
                height: frame.height as u16,
                rle_data: chunk.clone(),
            }),
        });
    }

    // 5. END — 结束 Display Set
    segments.push(Segment {
        segment_type: SegmentType::End,
        pts,
        dts: pts,
        payload: SegmentPayload::End,
    });

    segments
}

7.6 多窗口支持

Blu-ray 规范: 一个 Display Set 最多可以包含 2 个对象和 2 个窗口。当两条字幕在同一时间点出现但位置不同时（如双语字幕），需要使用多窗口机制。 
/// 多窗口场景处理
pub fn handle_multi_subtitle_frame(
    events: &[&Event],
    time: Timestamp,
    config: &RenderConfig,
) -> Vec<Segment> {
    // 最多 2 个对象（Blu-ray 限制）
    let objects: Vec<_> = events.iter().take(2).collect();

    if objects.len() == 1 {
        // 单字幕 — 单窗口
        build_display_set_single(objects[0], time, config)
    } else {
        // 双字幕 — 双窗口
        build_display_set_dual(objects[0], objects[1], time, config)
    }
}
