2.1 SUP/PGS 解码器

目标

实现一个轻量级 PGS 解码器，用于验证 ass2sup 输出的 SUP 文件是否合法。这是端到端验证的核心基础设施。

实现规格

组件函数名说明
段解析parse_segment(data: &[u8]) -> Option<ParsedSegment>从二进制数据解析单个 PGS 段
PCS 解析parse_pcs(payload: &[u8]) -> PcsInfo解析宽高、帧率、合成状态、对象列表
WDS 解析parse_wds(payload: &[u8]) -> Vec<WindowInfo>解析窗口定义
PDS 解析parse_pds(payload: &[u8]) -> PaletteInfo解析调色板
ODS 解析parse_ods(payload: &[u8]) -> ObjectInfo解析对象尺寸 + RLE 数据
RLE 解码rle_decode(data: &[u8], w: u16, h: u16) -> Vec<u8>PGS 特定 RLE 解码 (与编码器 rle.rs 对应)
完整验证verify_sup(data: &[u8]) -> VerifyResult验证整个 SUP 文件结构

验证检查项

  • 每段以 "PG" 魔数开头 (0x5047)
  • PCS 段在每个显示集的开头
  • WDS 在 PCS 之后、PDS 之前
  • ODS 在 PDS 之后
  • END 在每个显示集的末尾
  • PTS 单调递增
  • DTS ≤ PTS
  • 对象宽高 ≠ 0
  • 调色板条目数 ≤ 256
  • RLE 数据可以正确解码
  • 解码后的位图尺寸与 ODS 声明一致

🔧 3. P1: Bug 修复 (6 个已知 Bug)

3.1 B.1: \t 解析器忽略 t2

症状

\t(t1,t2,accel,tag) 解析后 t2 = t1，导致动画时间范围错误。

根因

event.rs:297 行: let t2 = t1; — 应为 let t2 = parts[2].parse()?;

修复方案

// event.rs — parse_single_tag(), \t branch
("t", rest) => {
    let parts: Vec<&str> = rest.split(',').collect();
    if parts.len() >= 4 {
        let t1: u64 = parts[0].parse().map_err(|e| 
            ParseError::InvalidEvent(format!("invalid t1: {}", e)))?;
        // BUG FIX: was `let t2 = t1;`
        let t2: u64 = parts[1].parse().map_err(|e| 
            ParseError::InvalidEvent(format!("invalid t2: {}", e)))?;
        let accel: f64 = parts[2].parse().unwrap_or(1.0);
        let tag = parts[3..].join(",");
        OverrideTag::Transform { tag, t1, t2, accel }
    } else {
        OverrideTag::Unknown(raw.to_string())
    }
}

验证测试

#[test]
fn test_t_tag_uses_distinct_t1_t2() {
    let event = Event::parse_from_line(
        EventType::Dialogue,
        "0,0:00:01.00,0:00:05.00,Default,,0,0,0,,{\\t(100,300,1.0,fs40)}"
    );
    if let Some(OverrideTag::Transform { t1, t2, .. }) = event.override_tags.first() {
        assert_eq!(*t1, 100);
        assert_eq!(*t2, 300);  // was 100 before fix
    }
}

3.2 B.2: \frx/\fry 旋转轴处理错误

症状

\frx (X 轴旋转) 和 \fry (Y 轴旋转) 在 renderer.rs 中被设置为 ctx.rotation (Z 轴旋转)，而非独立的旋转轴字段。

根因

RenderContext 没有独立的 frx/fry 字段，build_context 中:\nOverrideTag::Rotation { x, y, z } => { ctx.rotation = z as f32; }\n忽略了 x 和 y 分量。

修复方案

  1. 在 RenderContext 中添加 rotation_x: f32 和 rotation_y: f32 字段
  2. 在 build_context 中分别设置三个旋转轴
  3. 在 render_event 中，按 Z→Y→X 顺序应用旋转 (或合并为单一旋转矩阵)
// context.rs — RenderContext 新增字段
pub rotation_x: f32,  // \frx — X 轴旋转
pub rotation_y: f32,  // \fry — Y 轴旋转
pub rotation_z: f32,  // \frz — Z 轴旋转 (原有 rotation 字段)

// renderer.rs — build_context
OverrideTag::Rotation { x, y, z } => {
    ctx.rotation_x = x;
    ctx.rotation_y = y;
    ctx.rotation_z = z;
}
OverrideTag::RotationX(v) => ctx.rotation_x = v;
OverrideTag::RotationY(v) => ctx.rotation_y = v;
OverrideTag::RotationZ(v) => ctx.rotation_z = v;

// transform.rs — AffineTransform 新增
pub fn rotate_at_center(w: f32, h: f32, angle: f32) -> AffineTransform {
    // 围绕图像中心旋转
    AffineTransform::translate(w / 2.0, h / 2.0)
        .then(&AffineTransform::rotate(angle))
        .then(&AffineTransform::translate(-w / 2.0, -h / 2.0))
}

// renderer.rs — render_event 中应用
if ctx.rotation_z != 0.0 {
    let center_x = w as f32 / 2.0;
    let center_y = h as f32 / 2.0;
    let transform = AffineTransform::rotate_at(center_x, center_y, ctx.rotation_z.to_radians());
    let transformed = transform.apply_to_pixmap(&data, w, h);
}

3.4 B.5: \q=3 (换行样式 3) 与 \q=0 相同

症状

wrap_style 3 (不换行) 被当作 wrap_style 0 (智能换行) 处理。

ASS 换行样式规范

值名称行为
0智能换行在合适位置自动换行
1行尾换行只在 \N 处换行, 行尾自动截断
2不自动换行不换行, 超出部分截断
3行首换行只在 \N 处换行, 行首自动截断

修复方案

// renderer.rs — wrap_text
fn wrap_text(text: &str, ctx: &RenderContext, available_width: f32, shaper: &Shaper, font_id: ID, font_size: f32) -> Vec {
    match ctx.wrap_style {
        0 => smart_wrap(text, available_width, shaper, font_id, font_size),
        1 => wrap_at_newline_only(text, available_width, WrapEnd),  // \N only, truncate end
        2 => no_wrap(text),  // no wrapping at all
        3 => wrap_at_newline_only(text, available_width, WrapStart), // \N only, truncate start
        _ => smart_wrap(text, available_width, shaper, font_id, font_size),
    }
}

pub fn render_events_parallel(
    &self, events: &[&Event], timestamp_ms: u64
) -> Vec {
    // 1. 按空间位置分组 (不重叠的事件可以并行)
    let groups = group_non_overlapping(events);

    groups.par_iter().map(|group| {
        group.iter().map(|e| self.render_event(e, timestamp_ms))
    }).collect()
}

    🛡️ 6. P2: 错误恢复增强

    6.1 parse_lenient 增强

    目标

    当前 parse_lenient 从某些错误中恢复，但仍有改进空间。目标：尽可能多地从真实世界的不规范 ASS 文件中恢复。

    增强恢复点

    错误类型当前行为目标行为
    无效时间码返回 Err跳过该事件, 继续
    缺少字段返回 Err使用默认值填充
    未知 section忽略保留在 ScriptInfo.extra
    样式行字段数 < 23返回 Err截断/填充到 23
    事件行字段数 < 10返回 Err使用空字符串填充
    无效 OverrideTag静默忽略记录为 Unknown, 继续
    无效颜色值返回 Err使用黑色 (AssColor::default)
    无效数字值返回 Err使用 0 或默认值
    UTF-8 解码失败返回 Err尝试 latin-1 解码

    修复方案

    // lib.rs — parse_lenient 增强
fn parse_lenient(content: &str) -> (AssFile, Vec) {
    let mut errors = Vec::new();
    let mut events = Vec::new();
    
    for (line_num, line) in content.lines().enumerate() {
        if line.starts_with("Dialogue:") {
            match parse_event_line(line) {
                Ok(event) => events.push(event),
                Err(e) => {
                    errors.push(ParseError::with_context(e.clone(), line_num, line));
                    // 尝试修复
                    if let Some(fixed) = try_fix_event(line, &e) {
                        events.push(fixed);
                    }
                    // 继续解析, 不中断
                }
            }
        }
    }
    
    (ass_file, errors)
}

// 修复策略
fn try_fix_event(line: &str, error: &ParseError) -> Option {
    match error {
        ParseError::InvalidTimestamp(_) => {
            // 尝试宽松解析时间码 (允许缺少前导零)
            parse_timestamp_lenient(line)
        }
        ParseError::InvalidStyle(_) => None,  // 无法修复
        _ => None,
    }
}

    6.2 渲染错误降级

    规格

    • 字体缺失: 使用 sans-serif 回退，记录 warning
    • 字形缺失: 使用 tofu 字符 (□) 替代，不 panic
    • 内存不足: 分块渲染，每次处理 N 个事件
    • 损坏帧: 跳过并记录，继续下一帧
