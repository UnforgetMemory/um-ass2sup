# Sub-6: GPU 加速 (GPU Acceleration)

**Sprint**: S5
**周期**: 2-3 周
**依赖**: Sub-4
**阻塞**: 无（独立可优化）

## 目标

为大型 ASS 文件（> 5000 事件）提供 10x 渲染加速，通过 wgpu/vello GPU 后端实现。

## 范围

### In Scope
- 评估 wgpu vs vello vs 自研（benchmark 决定）
- 实现 `RendererBackend` trait
- 统一 CPU/GPU 接口
- 混合 pipeline：小任务 CPU，大任务 GPU
- 性能基准测试（criterion）

### Out of Scope
- 颜色管线（已属于 Sub-5）
- 字体子集化（属于 Sub-7）

## 架构决策

### Backend trait
```rust
pub trait RendererBackend: Send + Sync {
    fn draw_glyph(&mut self, glyph: &Glyph, pos: Point, color: Color);
    fn fill_rect(&mut self, rect: Rect, color: Color);
    fn apply_effect(&mut self, effect: &Effect);
    fn finalize(&mut self) -> RenderedFrame;
}
```

### 实现选择：vello
**理由**：
- 纯 Rust（无 C 依赖）
- 现代 GPU API（wgpu 兼容）
- Linebender 团队维护
- 支持 subpixel + hinting
- MSRV 1.88（与项目一致）

**风险**：
- vello 在 Linux 某些 GPU 驱动有 bug → 备选 CPU
- ABI 兼容 → 条件编译

### 混合策略
- 任务 < 100 事件 → CPU
- 任务 ≥ 100 事件 → GPU
- 任务 ≥ 5000 事件 → GPU 强制
