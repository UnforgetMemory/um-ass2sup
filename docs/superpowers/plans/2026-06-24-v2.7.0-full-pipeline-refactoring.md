# v2.7.0 全链路重构计划 — ass-core 接入 + CLI DDD 重写 + 旧代码清理

> **面向 AI 代理的工作者：** 必需子技能：使用 `subagent-driven-development`（推荐）逐任务实现此计划。
> 步骤使用复选框（`- [ ]`）语法来跟踪进度。
> **TDD 强制**。每个任务必须先写测试（RED），后写实现（GREEN），再人工验证（SURFACE）。

**目标**：全链路适配上游重构（ass-core、cosmic-text、ColorPipeline、DDD pgs-encoder），
将 `ass2sup-cli` 重写为 DDD 风格模块化架构，删除所有旧代码/归档。

**架构**：6 Wave 渐进式迁移

**技术栈**：Rust 2021, clap 4.x, thiserror, tracing, rayon
- **解析器**：`ass-core`（替代旧 `ass-parser`）
- **字体**：`cosmic-text`（唯一后端，移除 feature gate）
- **量化**：`ColorPipeline`（已重构）
- **编码**：DDD `pgs-encoder`（domain/encoding/decoding）
- **CLI 架构**：DDD 风格（cli/ + pipeline/ + config/）

## Wave 3：ass2sup-cli DDD 重写（核心）

### 目标模块结构

```
crates/ass2sup-cli/src/
│
├── main.rs               ← 入口：Args::parse() → run()
│
├── lib.rs                ← 库根：模块声明 + run() 函数 + 辅助函数
│
├── cli/                  ← 驱动适配层（Driver Adapter）
│   ├── mod.rs
│   ├── args.rs           ← Args + Resolution + FontMap
│   └── progress.rs       ← 进度条
│
├── pipeline/             ← 应用服务层（Application Service）
│   ├── mod.rs            ← PipelineStats, convert_file(), convert_to_bdn()
│   ├── batch.rs          ← 批量转换
│   ├── srt.rs            ← SRT 降级
│   └── check.rs          ← --check 验证模式
│
├── config/               ← 领域层（Domain）
│   ├── mod.rs            ← Config 结构体 + builder
│   ├── resolution.rs     ← 分辨率解析
│   ├── color_space.rs    ← 色彩空间
│   └── font.rs           ← FontMap + 字体检查
│
├── error.rs              ← CliError + Result 类型
├── telemetry.rs          ← tracing 初始化
└── ocr.rs                ← PaddleOCR 适配器
```

### 关键设计

```rust
// === config/ 统一配置（取代 args 直接传递） ===
pub struct Config {
    pub resolution: Resolution,
    pub fps: f64,
    pub max_colors: usize,
    pub dither: DitherMethod,
    pub color_space: ColorSpace,
    pub tonemap: Option<ToneMapOperator>,
    pub font: FontConfig,
    pub output: OutputConfig,
    pub parallel: ParallelConfig,
}

impl Config {
    pub fn from_args(args: &Args, doc: &SubtitleDocument) -> Result<Self>;
}

// === pipeline/ 方法复用（解决 convert_file 和 convert_to_bdn 重复） ===
impl PipelineService {
    /// 共享的渲染+量化逻辑
    fn render_and_quantize(&self, doc: &SubtitleDocument) -> Vec<QuantizedFrame>;

    /// SUP 编码路径
    fn encode_sup(&self, frames: &[QuantizedFrame]) -> Vec<u8>;

    /// BDN XML 路径
    fn encode_bdn(&self, frames: &[QuantizedFrame], stem: &str) -> Result<Vec<PathBuf>>;
}
```
