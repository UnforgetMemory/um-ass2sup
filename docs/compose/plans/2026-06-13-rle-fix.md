# RLE Encoder/Decoder Fix Plan

> **Goal**: Fix RLE data that decodes to more pixels than w×h, causing PotPlayer buffer overflow crash.

## Root Cause

Encoder produces `encode_run(color, 1, transparent_index)` for single opaque pixels:
```rust
// encoder.rs:72-74
} else if length == 1 && !color_in_collision_range {
    output.push(color);
    output.push(0x40);
}
```

The decoder (rle.rs:148) interprets `b & 0xC0 == 0x40` as transparent long run header, not as opaque color + marker. Result: decoder produces `color * 256 + 64` transparent pixels instead of 1 opaque pixel.

## Fix

Change encoder single-pixel format from `[color, 0x40]` to `[color, 0x01]`:
- `0x01` after color byte = single opaque pixel marker
- Decoder's `b & 0xC0 == 0x40` check won't match `0x01`

This requires updating both encoder (rle.rs) and verifying decoder handles `0x01` correctly.

- `crates/pgs-encoder/src/rle.rs:72-74` — encoder fix
- `crates/pgs-encoder/src/rle.rs:148-176` — verify decoder handles single pixel marker
